# 服务器、域名与上线方案说明

最后更新：2026-05-30

## 1. 当前状态

- 域名已经购买
- 服务器还未购买
- 域名解析还未配置
- ICP 备案还未开始
- 当前目标不是做高并发商业网站，而是稳定支撑本研究的正式实验运行

## 2. 对本实验的理解

本实验是一个需要多人在线协作的研究平台，核心特点如下：

- 被试会同时在线使用系统
- 前端需要稳定加载材料、工作台、AI 区和副线任务区
- 后端需要稳定保存实验进度、草稿、行为日志、问卷结果和任务状态
- 数据不能轻易丢失
- 实验当天不能因为部署、数据库或文件服务问题导致整体中断

虽然材料量本身大概率不算特别大，但系统真正的风险不在“总容量”本身，而在：

- 多人并发访问
- 数据库存储和恢复
- 材料文件与业务服务混在一起导致的资源争抢
- 实验期间故障后的恢复难度

因此，本项目在部署上应优先追求“稳”，而不是单纯追求最低成本。

## 3. 基本概念

### 3.1 服务器

服务器就是运行网站、后端程序和相关服务的远程电脑。

### 3.2 域名

域名就是用户访问网站时看到的名字，例如 `aiseek.tech`。

### 3.3 解析

解析就是把域名指向服务器公网 IP，让浏览器知道这个域名对应哪台服务器。

### 3.4 SSL / HTTPS

SSL 证书用于让网站从 `http` 变成 `https`，保证访问链路加密。

### 3.5 ICP 备案

如果服务器放在中国大陆，网站要公开上线，通常需要走 ICP 备案流程。

## 4. 当前建议方案

本项目当前采用方案 A：

### 4.1 云厂商与地域

- 云厂商：腾讯云
- 地域：腾讯云中国内地
- 推荐地域：上海或南京

原因：

- 便于走腾讯云备案
- 国内访问更稳定
- 对本研究的实际使用场景更友好

### 4.2 服务器

推荐购买：

- 1 台腾讯云 CVM
- 配置：`4 核 8G`
- 系统盘：`100GB SSD`
- 公网带宽：`10~20 Mbps`

说明：

- `4 核 8G` 是当前“稳妥与成本”之间更平衡的选择
- `8 核 16G` 不是不能买，但对当前阶段大概率偏重
- `5 Mbps` 可以跑，但多人同时在线时不够安心
- 如果预算允许，优先把带宽拉到 `10 Mbps` 以上

### 4.3 数据库

推荐购买：

- 1 个托管 PostgreSQL 数据库

原因：

- 数据库存实验核心数据，稳定性要求高
- 托管数据库更适合研究系统的正式运行
- 可减少自建数据库带来的备份、恢复、监控和运维风险
- 相比把所有东西都堆在一台 CVM 上，托管数据库更稳

当前判断：

- 如果只是本地开发或临时 demo，可以自建数据库
- 但如果目标是正式实验上线，推荐直接上托管 PostgreSQL

### 4.4 对象存储

推荐购买：

- 1 个腾讯云 COS

原因：

- 适合存放公司材料、上传文件、截图、导出文件等静态资源
- 能把文件存储和业务服务拆开
- 后期迁移服务器更容易
- 更适合长期维护研究材料

当前判断：

- COS 对当前研究是推荐项，不是因为容量一定巨大，而是因为“文件与业务解耦”更稳

### 4.5 域名

- 当前已购买域名，可继续使用
- 不需要因为 `.com` 更常见就重新买一个域名
- 当前重点是后续解析、备案和上线流程打通

## 5. 当前不建议购买的项

### 5.1 DNS 解析专业版

当前不建议购买。

原因：

- 先用腾讯云 / DNSPod 的基础解析即可
- 当前关键问题不是高级 DNS 能力，而是先把解析、部署和备案流程跑通

### 5.2 付费 SSL 证书

当前不建议购买。

原因：

- 前期可以先使用免费 SSL 方案
- 当前更重要的是服务器、备案、数据库和对象存储的稳定性

## 6. 推荐采购清单

建议当前采购为：

- 腾讯云中国内地 `1 台 CVM`
- `4 核 8G`
- `100GB SSD`
- `10~20 Mbps` 带宽
- `1 个托管 PostgreSQL`
- `1 个 COS`
- 当前域名继续使用

不建议当前购买：

- DNS 解析专业版
- 付费 SSL 证书

## 7. 推荐上线顺序

### 第一步：购买服务器

- 先完成腾讯云 CVM 购买

### 第二步：准备备案

- 因为服务器放在中国大陆，先启动腾讯云备案流程

### 第三步：配置解析

- 将域名解析到服务器公网 IP

### 第四步：部署系统

- 部署前端、后端、数据库连接和基础运行环境

### 第五步：接入 HTTPS

- 使用免费 SSL 方案先完成 HTTPS

### 第六步：实验前联调与演练

- 用真实流程完成一次完整实验彩排
- 重点检查：
  - 登录与分组
  - 指导语 / 测试题 / 测试轮 / 正式轮
  - 数据库存储
  - 材料访问
  - 副线任务
  - AI 区
  - HTTPS

## 8. 最终结论

对本研究当前阶段来说，最推荐的部署方案不是把 CVM 一味买大，而是：

- 服务器保持中等稳妥配置
- 数据库用托管版
- 文件材料上 COS
- 带宽不要买得太小

因此，当前最终建议为：

- `腾讯云中国内地`
- `1 台 CVM（4 核 8G / 100GB SSD / 10~20 Mbps）`
- `1 个托管 PostgreSQL`
- `1 个 COS`
- `当前域名继续使用`
- `先备案，再解析，再部署，再 HTTPS`
