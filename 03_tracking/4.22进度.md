# 4.22进度

## 一、项目整体流程是怎么打算的（按时间轴理解）

### 阶段 0：启动与对齐

- 收集原始材料
- 搭项目目录
- 明确这是长期全栈项目，不按普通零散任务推进
- 建立规则文件、规格文件、进度文件

### 阶段 1：文档与工程地基

- 建 monorepo
- 定技术方案：Next.js + NestJS + PostgreSQL
- 补规则、规格、跟踪体系
- 让项目从“聊天想法”变成“可追踪工程”

### 阶段 2：最小链路闭环

- 打通 session 创建 / 加入 / 查询
- 打通 waiting-room / instruction / A / B 页面骨架
- 打通 progress 写入、读取与 inspector 查看
- 让实验最小业务流先真实跑起来

### 阶段 3：A/B 工作台产品化

- 把 A/B 页面从纯调试骨架升级成工作台
- 形成顶部状态条 + 副线区 + 左材料区 + 右侧任务/AI 区的结构
- 让页面开始接近正式产品形态

### 阶段 4：副线与 AI 真实化

- 副线从静态卡片走向调度/打标/留存/回流
- AI 从占位接口走向真实 provider 接入
- 管理员页开始承担调试与参数控制职责

### 阶段 5：稳定化与验收

- 手工验收
- 继续消灭 UI/交互/环境类 bug
- 让规格、实现、运行态重新一致
- 在现有基础上进入下一阶段扩展
