# INTERROGATION_QUESTIONS.md

> 用于每次开始新项目 / 新模块 / 新功能前的审问清单。

---

## 1. 产品目标
- 这个产品/功能到底是给谁用的？
- 用户最核心的动作是什么？
- 用户完成这个动作后，理想结果是什么？
- 这个功能解决的核心问题是什么？
- 为什么现在要做它？

## 2. 范围定义
- 这次版本必须做什么？
- 这次版本明确不做什么？
- 有没有容易让人误以为要做、但其实先不做的东西？
- 最小可用版本（MVP）是什么？

## 3. 页面与流程
- 会涉及哪些页面？
- 用户会按什么顺序进入这些页面？
- 每个页面的核心任务是什么？
- 页面之间如何跳转？
- 成功流程是什么？
- 失败流程是什么？
- 空状态、加载状态、异常状态是什么？

## 4. 数据与后端
- 需要保存哪些数据？
- 需要展示哪些数据？
- 这些数据是谁创建的、谁修改的、谁查看的？
- 是否需要用户系统？
- 是否需要数据库？
- 是否需要文件上传？
- 是否需要权限区分？
- 哪些字段是必填？

## 5. 前端与体验
- 用户主要在手机上用还是电脑上用？
- 是否必须移动端优先？
- 有没有参考的 UI 风格？
- 更强调 UI 漂亮，还是 UX 清晰顺滑？
- 哪些交互一定要有反馈？

## 6. 技术选择
- 前端想用什么？
- 后端想用什么？
- 数据库想用什么？
- 部署想用什么？
- 是否允许引入第三方服务？
- 哪些依赖是希望尽量少引入的？

## 7. 验收标准
- 做到什么程度算“完成”？
- 哪些场景必须通过？
- 哪些 bug 可以接受留到下一阶段？
- 哪些 bug 绝对不能带着上线？

## 8. 项目现实约束
- 你当前有哪些不会的地方？
- 哪些部分是边做边学也能接受的？
- 哪些部分一旦做错代价会很高？
- 有没有时间、预算、资源上的限制？

---

## 默认使用方式
每次开始前，你可以直接对我说：

> “按 `INTERROGATION_QUESTIONS.md` 把我问透，先不要写代码。”
