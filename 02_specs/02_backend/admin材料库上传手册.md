# admin 材料库上传手册

这份手册对应当前已经落地的题库系统实现。

适用场景：

- 你要新增一个公司/案例
- 你要批量维护题库
- 你要让系统自动识别参与者材料与研究者材料
- 你要指定哪份研究者材料作为“系统自动填充源”

---

## 1. 先说最短路径

当前系统默认扫描：

`E:\Own_program\multi cooperation\00_start_materials\原始材料`

只要你把每个案例放成这个目录下的一个**子文件夹**，然后进入 `/admin -> 材料管理 -> 自动导入题库目录`，系统就会尝试自动识别并导入。

---

## 2. 推荐目录结构

强烈推荐以后统一按这个结构放：

```text
00_start_materials/
  原始材料/
    P01/
      case.json
      participant/
        1.公司简介（文）.txt
        2.财务摘要与关键指标.docx
        3.行业链与竞争格局（文或图）.docx
        4.内部访谈纪要（文或图）.docx
        5.外部访谈纪要（文或图）.docx
        6.生产与质量情况（文或图）.docx
        7.社会舆情与法务（文或图）.docx
        8.客户与订单概况（文）.docx
        688110-2025.pdf
      research/
        0.信息点记录（研究者用）.txt
```

每个案例一个文件夹，例如：

- `P01/`
- `P02/`
- `医疗器械-案例A/`

---

## 3. 系统如何自动识别

当前自动识别规则如下。

### 规则 A：每个子文件夹就是一个案例

例如：

```text
原始材料/
  P01/
  P02/
  P03/
```

系统会把 `P01`、`P02`、`P03` 分别识别成 3 个案例。

### 规则 B：如果存在 `participant/` 与 `research/` 子目录，优先按子目录分流

- `participant/` 里的文件：参与者可见材料
- `research/` 里的文件：研究者材料，不会进入参与者材料区

### 规则 C：如果没有 `participant/` / `research/` 分层，系统会按文件名猜测

只要文件名或相对路径里包含这些词，就会被当作研究者材料：

- `研究者用`
- `信息点记录`
- `答案`
- `research`

其他能识别的文件，会默认当成参与者材料。

### 规则 D：自动填充源优先找研究者材料

系统会优先识别：

- 文件名包含 `信息点记录`
- 或文件名包含 `研究者用`

的 `.txt` 文件，作为自动填充源。

---

## 4. `case.json` 怎么写

`case.json` 是可选的，但推荐使用。

它可以让你明确指定：

- 案例编号
- 公司名称
- 行业
- 简介
- 标签
- 显示顺序
- 参与者材料目录名
- 研究者材料目录名
- 自动填充源路径

示例：

```json
{
  "caseCode": "P01",
  "companyName": "高端装备样本 A",
  "roundLabel": "P01",
  "sector": "高端装备",
  "summary": "用于尽调员—投资经理协作实验的高端装备案例。",
  "tags": ["P01", "高端装备", "正式案例"],
  "participantDir": "participant",
  "researchDir": "research",
  "autoFillSource": "research/0.信息点记录（研究者用）.txt",
  "sortOrder": 1
}
```

### 字段说明

- `caseCode`
  - 用于案例编号，推荐唯一
- `companyName`
  - admin 中显示的案例名称
- `roundLabel`
  - 当前项目里常用的案例标签
- `sector`
  - 行业
- `summary`
  - 案例简介
- `tags`
  - 标签数组
- `participantDir`
  - 参与者材料目录名
- `researchDir`
  - 研究者材料目录名
- `autoFillSource`
  - 相对于案例目录的研究者源文件路径
- `sortOrder`
  - admin 中的排序顺序

---

## 5. 当前支持的文件类型

当前系统支持：

- `.txt`
- `.docx`
- `.pdf`
- `.xlsx`
- `.xls`

对应阅读器：

- `txt`：原生文本阅读器
- `docx`：`docx-preview`
- `pdf`：PDF 阅读器
- `xlsx/xls`：表格阅读器

---

## 6. 研究者材料一定不要给参与者看

这件事系统已经在做保护，但你还是要遵守命名纪律。

推荐把以下内容放在 `research/`：

- `0.信息点记录（研究者用）.txt`
- 标准答案
- 评分底稿
- 研究者提示

这些材料**不会显示在参与者材料区**。

---

## 7. 兼容旧结构

当前系统兼容旧的扁平结构，比如现在的 `P01`：

```text
P01/
  0.信息点记录（研究者用）.txt
  1.公司简介（文）.txt
  2.财务摘要与关键指标.docx
  ...
```

这种结构也能导入。

但长期建议还是改成：

```text
P01/
  participant/
  research/
  case.json
```

因为：

- 更不容易误把研究者材料暴露给参与者
- 更适合批量维护
- 更便于以后扩展评分、标签、版本

## 8. admin 中现在有哪些动作

进入 `/admin -> 材料管理` 后，你可以做：

1. 自动导入题库目录
2. 导入当前基线 P01
3. 新建/编辑公司信息
4. 上传单个材料
5. 替换单个材料
6. 删除单个材料
7. 调整材料顺序
8. 指定自动填充源
9. 预览材料显示效果

---

## 9. 推荐工作流

### 工作流 A：新增一整个案例

1. 在 `00_start_materials/原始材料/` 下新建一个案例目录
2. 放入 `participant/` 与 `research/`
3. 写好 `case.json`
4. 打开 admin
5. 点击“自动导入题库目录”
6. 在 admin 中检查显示与预览效果

### 工作流 B：只替换某个材料文件

1. 可以直接在 admin 中点“替换”
2. 也可以先改本地题库目录，再点“自动导入题库目录”整体刷新

如果你本次修改的是：

- 案例名称
- 行业
- 简介
- 材料全量结构

更推荐走“改目录 + 自动导入”。

---

## 10. 当前实现边界

当前自动导入系统已经能直接用，但有几个现实边界要知道：

1. 它默认从固定目录扫描，不是任意选择文件夹
2. `autoFillSource` 当前最适合 `.txt` 研究者材料
3. 自动解析公司信息依赖研究者文本里存在较规范字段
4. 如果 `case.json` 缺失，系统会尽量按文件夹名自动补默认值

---

## 11. 最佳实践

建议以后每个案例都遵守这几点：

1. 一个案例一个目录
2. 参与者材料和研究者材料分目录
3. 一定写 `case.json`
4. 研究者文本命名清楚，不要混在参与者材料里
5. 文件名前尽量带顺序号，方便材料区 tab 顺序稳定

推荐命名：

- `1.公司简介（文）.txt`
- `2.财务摘要与关键指标.docx`
- `3.行业链与竞争格局（文或图）.docx`

而不是：

- `最新版最终修订财务材料真的最终版.docx`

---

## 12. 一句话总结

以后你最省心的方式就是：

**把案例按规范文件夹放进 `00_start_materials/原始材料/`，再到 admin 里点“自动导入题库目录”。**
