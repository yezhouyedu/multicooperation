# 部署运行手册

> 状态：2026-06-06 第一版 P0
> 范围：Docker 单机部署，先跑通裸 IP，再接域名和 HTTPS。

## 1. 文件说明

- `02_specs/05_server_deploy/命令运行清单.md`：最重要的执行入口，集中列出你后续要运行的所有命令。
- `apps/web/Dockerfile`：构建并启动 Next.js 前端。
- `apps/server/Dockerfile`：构建并启动 NestJS 后端。
- `apps/server/docker-entrypoint.sh`：后端容器启动前执行 Prisma migration。
- `compose.production.yml`：生产式单机 compose。
- `.env.production.example`：生产环境变量模板。
- `.dockerignore`：减少 Docker build 上下文，并排除私钥和本地数据。
- `scripts/deploy/check-server.ps1`：本地通过 SSH 检查服务器状态。
- `scripts/deploy/fix-ssh-key-permissions.ps1`：修复 Windows OpenSSH 私钥权限太开放的问题。
- `scripts/deploy/probe-ssh-users.ps1`：私钥权限正常但 publickey 登录失败时，自动试探常见服务器用户名。
- `scripts/deploy/upload-project.ps1`：本地打包并上传项目到服务器。
- `scripts/deploy/prepare-server.sh`：服务器初始化 Docker 与基础目录。
- `scripts/deploy/deploy-prod.sh`：服务器上构建、启动并检查生产服务。
- `scripts/deploy/show-prod-logs.sh`：服务器上查看 compose 服务日志。

## 2. 推荐协作方式

部署和环境配置很慢，后续默认用带进度输出的脚本推进。

本地先执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/deploy/check-server.ps1
```

服务器初始化脚本、生产部署脚本都会输出类似：

```text
==> [1/8] Checking deployment files
==> [2/8] Showing compose plan
==> [3/8] Building images
```

如果卡住，优先把最后一个阶段标题和其后的错误日志贴回来。

## 3. 本地生产式验证

在根目录创建本地测试用 `.env.production`：

```powershell
Copy-Item .env.production.example .env.production
```

至少修改：

```text
POSTGRES_PASSWORD=一个长随机密码
NEXT_PUBLIC_SERVER_BASE_URL=http://localhost:3001
```

启动：

```powershell
docker compose --env-file .env.production -f compose.production.yml up -d --build
```

查看状态：

```powershell
docker compose --env-file .env.production -f compose.production.yml ps
docker compose --env-file .env.production -f compose.production.yml logs -f server
```

访问：

- web：`http://localhost:3000`
- server health：`http://localhost:3001/health`

停止：

```powershell
docker compose --env-file .env.production -f compose.production.yml down
```

注意：不要随意加 `-v`，否则会删除数据库和后端 storage volume。

## 4. 服务器 P0 部署步骤

服务器准备：

1. 安装 Docker 和 Docker Compose plugin。
2. 上传或拉取项目代码。
3. 在服务器项目根目录创建 `.env.production`。
4. 填入真实 `POSTGRES_PASSWORD`、AI 配置和 `NEXT_PUBLIC_SERVER_BASE_URL`。

### 4.1 本地检查服务器

```powershell
powershell -ExecutionPolicy Bypass -File scripts/deploy/check-server.ps1
```

默认使用：

- host：`49.233.203.108`
- user：`root`
- key：`E:\Own_program\multi_cooperation_secrets\ssh\first_try.pem`

如果服务器用户名不是 `root`：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/deploy/check-server.ps1 -User ubuntu
```

### 4.2 上传项目

```powershell
powershell -ExecutionPolicy Bypass -File scripts/deploy/upload-project.ps1
```

默认上传到：

```text
/opt/multi-cooperation
```

上传脚本会排除：

- `.git`
- `node_modules`
- 构建产物
- `.env*`
- `storage`
- 私钥文件

### 4.3 服务器初始化

SSH 登录服务器后执行：

```bash
cd /opt/multi-cooperation
bash scripts/deploy/prepare-server.sh
```

### 4.4 创建生产环境变量

```bash
cd /opt/multi-cooperation
cp .env.production.example .env.production
nano .env.production
```

至少修改：

```text
POSTGRES_PASSWORD=一个长随机密码
NEXT_PUBLIC_SERVER_BASE_URL=http://49.233.203.108:3001
```

AI 可先留空，但正式彩排前要补真实值：

```text
OPENAI_BASE_URL=
OPENAI_API_KEY=
OPENAI_MODEL=
```

### 4.5 启动生产服务

服务器上执行：

```bash
cd /opt/multi-cooperation
bash scripts/deploy/deploy-prod.sh
```

脚本会自动执行：

- compose 配置检查
- Docker build
- compose up
- server health 检查
- web 登录页检查

启动：

```bash
docker compose --env-file .env.production -f compose.production.yml up -d --build
```

检查：

```bash
docker compose --env-file .env.production -f compose.production.yml ps
docker compose --env-file .env.production -f compose.production.yml logs -f server
```

P0 访问地址：

- web：`http://49.233.203.108:3000`
- server health：`http://49.233.203.108:3001/health`

## 5. 数据持久化

当前 compose 使用两个 Docker volume：

- `postgres_data`：PostgreSQL 数据。
- `server_storage`：后端 `storage/`，包括材料、图片附件、导出包。

重要纪律：

- `docker compose down` 不会删除 volume。
- `docker compose down -v` 会删除 volume，正式环境不要执行。
- 迁移服务器前必须同时备份数据库和 `server_storage`。

## 6. 常用运维命令

查看容器：

```bash
docker compose --env-file .env.production -f compose.production.yml ps
```

看后端日志：

```bash
docker compose --env-file .env.production -f compose.production.yml logs -f server
```

或：

```bash
bash scripts/deploy/show-prod-logs.sh server
```

看数据库日志：

```bash
docker compose --env-file .env.production -f compose.production.yml logs -f postgres
```

重启后端：

```bash
docker compose --env-file .env.production -f compose.production.yml restart server
```

重新构建并启动：

```bash
docker compose --env-file .env.production -f compose.production.yml up -d --build
```

## 7. P1 收口提醒

裸 IP 跑通后，不要长期暴露 `3000 / 3001 / 5432`。

P1 应改为：

- 外部只开放 `80 / 443 / 22`。
- 用 Caddy 或 Nginx 做反向代理。
- PostgreSQL 不对公网开放。
- 前端 API 地址改成 HTTPS 域名。
