# 命令运行清单

> 状态：2026-06-06
> 用途：只回答“下一步我该运行哪些命令”。每一步都可以单独执行，卡住时把当前步骤标题和错误日志贴回来。

## 0. 运行位置说明

下面命令分两类：

- **本地 PowerShell**：在你自己的 Windows 电脑，项目根目录 `E:\Own_program\multi cooperation` 运行。
- **服务器 SSH 终端**：登录服务器后，在服务器上运行。

本地项目根目录：

```powershell
cd "E:\Own_program\multi cooperation"
```

SSH 私钥位置：

```text
E:\Own_program\multi_cooperation_secrets\ssh\first_try.pem
```

当前已确认服务器 SSH 用户名：

```text
ubuntu
```

服务器项目目录：

```text
/opt/multi-cooperation
```

---

## 执行纪律

后续按步骤逐个运行。每一步运行前由用户明确允许或自行执行，不默认连续执行下一步。

如果某一步失败，不继续下一步，先把当前步骤标题和错误日志贴回来。

---

## 1. 本地检查服务器

运行位置：本地 PowerShell。

如果第一次运行遇到：

```text
WARNING: UNPROTECTED PRIVATE KEY FILE
Permissions ... are too open
```

先修复私钥权限：

```powershell
cd "E:\Own_program\multi cooperation"
powershell -ExecutionPolicy Bypass -File scripts/deploy/fix-ssh-key-permissions.ps1
```

然后再运行服务器检查：

```powershell
cd "E:\Own_program\multi cooperation"
powershell -ExecutionPolicy Bypass -File scripts/deploy/check-server.ps1 -User ubuntu
```

如果权限已修好，但仍然看到：

```text
Permission denied (publickey)
```

说明服务器不接受“当前用户名 + 当前私钥”的组合。先自动试常见用户名：

```powershell
cd "E:\Own_program\multi cooperation"
powershell -ExecutionPolicy Bypass -File scripts/deploy/probe-ssh-users.ps1
```

如果重新探测出其他可用用户名，后续命令都要把 `ubuntu` 换成实际用户名。

```powershell
powershell -ExecutionPolicy Bypass -File scripts/deploy/check-server.ps1 -User ubuntu
powershell -ExecutionPolicy Bypass -File scripts/deploy/upload-project.ps1 -User ubuntu
```

如果所有用户名都失败，需要去云厂商控制台确认：

- 这台服务器绑定的是不是这把 key。
- 系统默认 SSH 用户名是什么。
- 是否禁用了 root 登录。
- 安全组是否允许 TCP 22。

正常会看到：

- SSH key 信息。
- `whoami`、`hostname`、`uname -a`。
- CPU、内存、磁盘。
- Docker 是否已安装。
- 当前监听端口。

## 2. 上传项目到服务器

运行位置：本地 PowerShell。

注意：如果看到前一步已经报错，但脚本仍在上传，先按 `Ctrl + C` 停止。当前脚本已改为远程目录创建失败时自动停止，并会在 `/opt/multi-cooperation` 权限不足时尝试 `sudo mkdir` 和 `sudo chown ubuntu`。

如果曾经看到 `rsync warning: some files vanished`，说明旧脚本把临时解压目录放在目标目录内部导致同步时互相影响；当前脚本已改为使用 `/tmp/multi-cooperation-deploy-new` 作为临时目录。重新运行本步骤即可。

```powershell
cd "E:\Own_program\multi cooperation"
powershell -ExecutionPolicy Bypass -File scripts/deploy/upload-project.ps1 -User ubuntu
```

正常会看到：

- 创建远端目录。
- 创建本地 tar 包。
- 上传到服务器。
- 解压到 `/opt/multi-cooperation`。

上传脚本会排除：

- `.git`
- `node_modules`
- 构建产物
- `.env*`
- `storage`
- 私钥文件

## 4. 服务器初始化

运行位置：服务器 SSH 终端。

```bash
cd /opt/multi-cooperation
bash scripts/deploy/prepare-server.sh
```

正常会看到：

- `[1/7] Detecting OS`
- `[2/7] Checking hardware`
- `[3/7] Installing basic tools`
- `[4/7] Installing Docker if needed`
- `[5/7] Enabling Docker`
- `[6/7] Creating app directory`
- `[7/7] Final server check`

如果这一步失败，把失败阶段标题和最后 30-80 行日志贴回来。

---

## 5. 创建生产环境变量

运行位置：服务器 SSH 终端。

```bash
cd /opt/multi-cooperation
cp .env.production.example .env.production
nano .env.production
```

至少要改：

```text
POSTGRES_PASSWORD=改成一个长随机密码
NEXT_PUBLIC_SERVER_BASE_URL=http://49.233.203.108:3001
```

AI 配置如果现在要接真实模型，也在这里填：

```text
OPENAI_BASE_URL=
OPENAI_API_KEY=
OPENAI_MODEL=
```

保存 nano：

- `Ctrl + O`
- 回车
- `Ctrl + X`

---

## 6. 启动生产服务

运行位置：服务器 SSH 终端。

```bash
cd /opt/multi-cooperation
bash scripts/deploy/deploy-prod.sh
```

正常会看到：

- `[1/8] Checking deployment files`
- `[2/8] Showing compose plan`
- `[3/8] Building images`
- `[4/8] Starting services`
- `[5/8] Waiting for containers`
- `[6/8] Checking server health`
- `[7/8] Checking web`
- `[8/8] Deployment summary`

这一步第一次会慢，因为要拉 Docker 镜像、安装依赖、构建 web/server。

---

## 7. 浏览器验证

运行位置：本地浏览器。

打开：

```text
http://49.233.203.108:3000/login
```

检查后端：

```text
http://49.233.203.108:3001/health
```

如果 3000 / 3001 打不开，先检查云服务器安全组是否放行：

- TCP 3000
- TCP 3001

P1 接域名 HTTPS 后会收口这些端口；P0 裸 IP 阶段先临时放行。

## 8. 查看运行状态

运行位置：服务器 SSH 终端。

```bash
cd /opt/multi-cooperation
docker compose --env-file .env.production -f compose.production.yml ps
```

查看后端日志：

```bash
cd /opt/multi-cooperation
bash scripts/deploy/show-prod-logs.sh server
```

查看前端日志：

```bash
cd /opt/multi-cooperation
bash scripts/deploy/show-prod-logs.sh web
```

查看数据库日志：

```bash
cd /opt/multi-cooperation
bash scripts/deploy/show-prod-logs.sh postgres
```

## 9. 重启服务

运行位置：服务器 SSH 终端。

重启后端：

```bash
cd /opt/multi-cooperation
docker compose --env-file .env.production -f compose.production.yml restart server
```

重启全部：

```bash
cd /opt/multi-cooperation
docker compose --env-file .env.production -f compose.production.yml restart
```

重新构建并启动：

```bash
cd /opt/multi-cooperation
bash scripts/deploy/deploy-prod.sh
```

---

## 10. 停止服务

运行位置：服务器 SSH 终端。

```bash
cd /opt/multi-cooperation
docker compose --env-file .env.production -f compose.production.yml down
```

不要执行：

```bash
docker compose down -v
```

因为 `-v` 会删除数据库和后端 storage volume。

## 12. 卡住时贴什么

如果卡住，优先贴：

1. 当前执行的是第几步，例如“第 6 步启动生产服务”。
2. 终端里最后一个阶段标题，例如 `[3/8] Building images`。
3. 最后 30-80 行错误日志。

如果是服务已经启动但网页打不开，贴：

```bash
cd /opt/multi-cooperation
docker compose --env-file .env.production -f compose.production.yml ps
docker compose --env-file .env.production -f compose.production.yml logs --tail=120 server
docker compose --env-file .env.production -f compose.production.yml logs --tail=120 web
```
