# 05_server_deploy

本目录专门放服务器上线与运维相关文档。

它和 `04_pre_deploy/` 的边界是：

- `04_pre_deploy/`：变量记录、导出包、数据库文件夹阅读、材料存储口径。
- `05_server_deploy/`：服务器、Docker、域名、HTTPS、SSH、备份恢复、日志排障、上线协作方式。

## 当前文件

- `命令运行清单.md`
  - 后续实际要运行的本地命令、SSH 命令、服务器命令都集中放在这里。
- `上线前工作指导.md`
  - 记录 P0/P1/P2 上线阶段、协作方式、SSH 私钥位置约定、正式实验前运维边界。
- `部署运行手册.md`
  - 记录 Docker compose 启停、生产环境变量、日志查看、裸 IP 验证和后续域名 HTTPS 收口。

## 当前原则

- 业务代码在本地仓库改，服务器只作为部署运行环境。
- SSH 私钥不进仓库、不进 Markdown、不放 `02_specs/`。
- 长耗时部署任务优先写成可观察脚本或分步骤命令，让操作者能看到进度。
- 生产部署前必须先在本地跑过生产式验证。
