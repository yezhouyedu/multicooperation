## 1. 本轮执行原则
- 先把规格文档收口成唯一真相来源，再按代码包推进。
- 每完成一个文档阶段或代码包，都必须更新 `03_tracking/progress.md`。
- `CHECKLIST_MANUAL.md` 暂不提前改成未来态，等真实代码跟上后再同步。
- 本轮不做 `截图` 功能，不做通用材料后台，不恢复旧的 B 等待页。

## 2. 已冻结的实验口径
- 管理员只上传合格手机号，不再预设 `A/B`。
- 进入顺序决定角色：奇数位 `A`，偶数位 `B`，相邻两人自动配对。
- 一组 `A/B` 共享同一条 session 内随机、无放回的固定公司顺序。
- `A` 每家公司有 `5` 分钟单公司窗口；`B` 可提前做自己的部分，但必须在 A 信息解锁且明确打开过 A 信息区后才能提交。
- 正式实验固定为 `3` 个工作段 + `2` 个休息问卷段，默认工作段 `20` 分钟、休息段 `5` 分钟。
- 顶栏倒计时对主工作台、测试轮、休息问卷页、副线展开页统一可见。
- 顶部副线入口固定文案“待处理事宜”，滚动文案固定“您有新事项入库，请尽快处理”，从右往左滚动，滚完等待 `30s` 后再次滚动。
- 测试轮独立存在，位于指导语后、正式实验前，日志与正式轮分开。
- AI 等级按工作段 `1/2/3` 配置为 `basic` 或 `advanced`；主线和副线在同一工作段内共用同一等级。
- 材料接入本轮只固定支持 `P01 + 两份正式表单`。

## 3. 代码包 1：后台入口与实验配置地基
- `/admin` 继续作为唯一后台入口；`/dev/session-inspector` 保持兼容跳转。
- admin 被试导入改为 `phone-only`，移除角色列、角色下拉和后端 `role` 入参依赖。
- 新增全局实验配置：
  - 工作段时长
  - 休息段时长
  - 工作段 `1/2/3` 的 AI 等级
  - 问卷模板
- 本包完成后，后台应能保存“名单 + 配置”，但还不要求主线 UI 已跟新流程贯通。

## 4. 代码包 2：自动配对、公司顺序、统一运行态
- 登录时按进入顺序自动分配 `A/B` 并完成 session 配对。
- session 启动时一次性生成随机、无放回的公司顺序；A/B 共用。
- 新增统一运行态接口，返回：
  - 当前阶段
  - 当前工作段索引
  - 当前阶段剩余秒数
  - 当前公司与公司序号
  - A 单公司剩余秒数
  - A 信息解锁状态
  - B 是否已查看 A 信息（行为记录）
  - B 是否可提交
  - 当前是否空窗、是否冻结、副线是否可用
- 测试轮与正式轮的任务状态、日志、快照分开记录。

## 5. 代码包 3：主线工作流重写
- A 端显示阶段倒计时与单公司 `5` 分钟倒计时。
- `A` 的 `5` 分钟到时自动保存并解锁该公司的 A 信息给 `B`。
- 若工作段先结束，则以工作段为优先：冻结当前公司，下一工作段恢复剩余公司时间。
- `/workspace/b` 成为唯一正式 B 工作台。
- B 空窗态保留主界面壳子，材料区 / A 信息区 / 作答区按规则置空或禁用，副线仍可做。
- B 在有公司时可立即看自有材料、写判断、使用主线 AI。
- A 信息解锁瞬间保存一次 B 端“5分钟快照 + 时间戳”。

## 6. 代码包 4：测试轮、段引擎、休息问卷段
- 在指导语后新增测试轮页面，前端先复用正式工作台结构。
- 固定实现 `3` 个工作段和 `2` 个休息问卷段。
- 工作段结束时冻结主线进度，下一工作段恢复。
- 休息段展示单选问卷页，并继续显示顶栏倒计时。
- 进入新工作段时清空副线累计完成进度，但保留主线进度与主线 AI。

## 7. 代码包 5：副线工作台与 AI 双上下文
- 顶部副线入口不再显示数量、详细政策标题、市场实时快讯。
- “待处理事宜”和滚动消息都可点击，进入同一个副线展开页。
- 副线展开页复用主工作台结构：左材料、右上作答、右下 AI。
- 题目切换 UX 为：答题后不自动跳题，通过“上一题 / 下一题”切换。
- 主线 AI 按“参与者 + 公司 + 当前轮次”隔离，切换公司时 UI 上下文清空。
- 副线 AI 按“参与者 + 当前实验 + 当前工作段”连续，不因切换公司清空。
- `basic` 隐藏上传按钮并拒绝图片，`advanced` 允许图片。

## 8. 代码包 6：真实材料与正式表单
- 材料区固定支持 `P01` 的 `txt/docx/pdf` 切换阅读。
- A 作答区完全按 `A端尽调表_参与者可见版v2.docx` 实现。
- B 作答区完全按 `B端投资判断表_参与者可见版.docx` 实现。
- `.txt` 直接渲染，`.docx` 走文本/结构化展示，`.pdf` 走嵌入式阅读器。
- 本包结束时，正式材料与正式表单应能跑通一轮真实任务。
