## 2. 已冻结的实验口径
- 管理员只上传合格手机号，不再预设参与者角色。
- 参与者先按进入顺序自动成组，再在组内随机分配 `A/B`；参与者可见文案统一使用 `A/B`，不再使用“尽调员 / 投资经理”。
- 同一组共享同一条 session 内随机、无放回、可复现的正式公司顺序。
- `A` 每家公司有 `5` 分钟单公司窗口；`B` 可提前做自己的部分，但只有在 A 信息解锁后才能提交；是否点击过 `A信息` 只做记录，不再作为提交门槛。
- 主流程当前固定为：指导语 → 同步准备页 1 → 测试题 → 测试轮 → 同步准备页 2 → 正式轮。
- 正式实验固定为 `3` 个工作段 + `2` 个休息问卷段，默认工作段 `20` 分钟、休息段 `5` 分钟。
- 顶栏倒计时对主工作台、测试轮、休息问卷页、副线展开页统一可见。
- 顶部任务2入口固定文案“任务2”，滚动文案固定“任务2有新题目，请及时处理”，从右往左滚动。
- 测试轮独立存在，位于测试题通过后、正式实验前；其任务、行为日志、草稿和正式轮分开记录。
- AI 等级按工作段 `1/2/3` 配置为 `basic` 或 `advanced`；任务1和任务2在同一工作段内共享同一等级。
- 材料系统已扩展为最终案例库导入 + 角色分层材料：正式 `P01-P36`，测试轮 `P37`。
- 任务2变量 A 当前操纵的是提醒频率，不是题目实际到达速率；后台实际到达统一按连续排期生成。

## 3. 代码包 1：后台入口与实验配置地基
- `/admin` 继续作为唯一后台入口；`/dev/session-inspector` 保持兼容跳转。
- admin 被试导入维持 `phone-only`，不回退到人工指定角色。
- 全局实验配置至少覆盖：
  - 工作段时长
  - 休息段时长
  - 测试轮时长
  - 工作段 `1/2/3` 的 AI 等级
  - 测试题模板
  - 休息问卷模板
- 本包完成后，后台应能保存“名单 + 配置 + 双问卷模板”，并为后续主流程提供统一配置源。

## 4. 代码包 2：自动配对、共享顺序、统一运行态
- 登录时先按进入顺序自动配对；配对完成后在组内随机决定 `A/B`。
- session 启动时一次性生成共享正式公司顺序；测试轮单独生成 practice assignment，不复用正式轮顺序。
- 统一 runtime 接口至少返回：
  - 当前阶段
  - 当前工作段索引
  - 当前阶段剩余秒数
  - 当前公司与公司序号
  - A 单公司剩余秒数
  - A 信息解锁状态
  - `bViewedAInfoAt` / 是否点击过 `A信息`
  - B 是否可提交
  - 当前是否空窗、是否冻结、副线是否可用
  - 测试题模板、测试题通过态、教学引导态（按阶段返回）
- 测试轮与正式轮的任务状态、日志、快照分开记录。

## 5. 代码包 3：主线工作流回正
- A 端显示阶段倒计时与单公司 `5` 分钟倒计时。
- `A` 的 `5` 分钟到时自动保存并解锁该公司的 A 信息给 `B`。
- 若工作段先结束，则以工作段结束优先：冻结当前公司；下一工作段恢复剩余进度。
- `/workspace/b` 作为唯一正式 B 工作台；`/workspace/b-waiting` 只保留兼容跳转，不再承担正式流程。
- B 有公司时可立即看自己的材料、写判断、使用主线 AI。
- A 信息解锁后，B 即可提交；是否点开 `A信息` tab 只做行为记录，不做门槛。
- A 信息解锁时保留一次 B 端关键快照与时间戳。

## 6. 代码包 4：测试题、测试轮、同步准备页、休息问卷
- 指导语后先进入 `practice_ready` 同步准备页；双方 ready 后进入测试题。
- 测试题通过前不进入测试轮；双方都通过后自动进入测试轮。
- 测试轮页面不再是独立壳子，而是导向真实工作台。
- 正式轮前再经过一次 `formal_ready` 同步准备页。
- 固定实现 `3` 个工作段和 `2` 个休息问卷段。
- 休息问卷页继续显示统一顶栏与阶段倒计时。

## 7. 代码包 5：副线工作台与 AI 双上下文
- 顶部副线入口不显示具体数量，不显示“市场实时快讯”。
- “任务2”和滚动消息都可点击，进入任务2展开页。
- 任务2展开页复用正式工作台布局：左材料、右上答题、右下 AI。
- 题目切换通过“上一题 / 下一题”实现；选择答案后给轻量反馈，但不自动跳题。
- 主线 AI 按“参与者 + 公司 + 当前轮次”隔离。
- 副线 AI 按“参与者 + 当前实验 + 当前工作段”连续。
- `basic` 隐藏图片能力；`advanced` 开放图片能力。
- 测试轮不生成真实 practice 任务2计划；教学引导需要点击任务2题目时，由前端临时显示 `practice_demo_sidetask` 演示题。

## 8. 代码包 6：真实材料与正式表单
- 材料区支持正式 / 测试轮双层案例库扫描；当前最终版要求正式 `P01-P36`、测试轮 `P37`，不再依赖 P01 fallback。
- A 作答区按 `A端尽调表_参与者可见版v4.docx` 落地。
- B 作答区按 `B端投资判断表_参与者可见版v3.docx` 落地。
- `.txt` 直接渲染，`.docx` 结构化展示，`.pdf` 嵌入阅读。
- 本包结束时，正式材料、测试轮材料、角色分层材料、主工作台表单与提交链路应能完整跑通。

6. 再验副线入口、副线展开页、提醒频率变量、AI 双上下文。
7. 最后验正式案例材料、角色分层材料、主工作台表单与正式提交链路。
