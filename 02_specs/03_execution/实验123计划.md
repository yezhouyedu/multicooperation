# 实验 1/2/3 模式切换计划

## 1. 目标

本项目继续使用同一套参与者页面和同一套正式实验流程，不复制三套实验页。

admin 同一时间只选择一种实验模式：

- `manual`：手动 / 通用
- `ai_upgrade`：实验 1，AI 能力升级随机化
- `side_reminder`：实验 2，副线提醒频率随机化
- `coop_narrative`：实验 3，合作叙事随机化

admin 切换实验模式只影响新创建的 session。旧 session 使用创建时保存的实验快照，不受后续 admin 修改影响。

## 2. 模式互斥与默认变量

实验 1/2/3 不能叠加启动。每个模式只随机自己的核心变量，其余变量使用 admin 模式卡片中的固定下拉。

默认固定变量按当前师兄口径：

- AI：基础版
- 副线提醒：continuous 高频提醒
- 叙事信息：neutral_info 中性信息

### manual

- 不启用实验 1/2/3 预设随机化
- 工作段 AI 使用原有段 1/2/3 手动配置
- 副线提醒和叙事沿用现有 session 初始化逻辑

### 实验 1：AI 能力升级

- 随机：`upgradeCohort = early_upgrade / late_upgrade`
- early：段 1 BASIC，段 2 ADVANCED，段 3 ADVANCED
- late：段 1 BASIC，段 2 BASIC，段 3 ADVANCED
- 固定：副线提醒、叙事信息
- 默认固定：`continuous`、`neutral_info`

### 实验 2：副线提醒频率

- 随机：`sideDispatchMode = continuous / batch`
- 固定：AI 能力、叙事信息
- 默认固定：`BASIC`、`neutral_info`

### 实验 3：合作叙事

- 随机：`narrativeGroup = coop_narrative / neutral_info`
- 合作叙事组再随机三主题顺序
- 固定：AI 能力、副线提醒
- 默认固定：`BASIC`、`continuous`

## 3. 运行态与留痕

新 session 创建时，后端把当时的 admin 配置固化为 session 级实验快照。

快照至少记录：

- 实验模式
- 随机变量结果
- 固定变量取值
- 三段 AI 状态
- 副线提醒模式
- 叙事组与主题顺序
- 对应随机 seed

任务级额外记录：

- A 处理该公司时的 AI 状态
- B PreA 阶段 AI 状态
- B PostA 阶段 AI 状态
- 是否跨 AI 升级边界

AI 请求级额外记录：

- 请求时 AI 状态
- 实际模型名
- 请求时是否允许图片上传

## 4. 前台提示

参与者前台统一显示 `A/B` 字母角色；不显示处理组、实验组等研究者标签。

AI 区始终显示功能状态 badge：

- 基础版
- 升级版

实验 1 中：

- AI 升级前一个休息页显示升级说明
- 升级后第一个工作界面显示一次升级提示
- B 查看尽调员信息时，显示尽调员处理该公司时使用的 AI 状态

指导语页面采用积木结构：

- 通用标题
- 通用说明
- 尽调员角色说明
- 投资经理角色说明
- 当前实验模式条件块

这些文案在 admin 中维护。

## 5. 验收

- admin 可选择四种模式
- 实验 1/2/3 模式互斥
- 模式卡片展示随机项和固定项
- 固定项可通过下拉调整
- 新 session 保存快照，旧 session 不被 admin 后续修改影响
- 导出包含 session 快照、随机化审计、AI 请求模型与任务级 AI 状态
- `corepack pnpm --filter server build` 通过
- `corepack pnpm --filter web build` 通过
## 2026-06-17 增补：段前指导语与实验 3 叙事匹配

三个正式工作段前新增“阅读材料”页。该页是流程阶段，不新增实验 1/2/3 之外的新处理变量。

- `manual`、实验 1 `ai_upgrade`、实验 2 `side_reminder`：固定展示 `neutral_1 → neutral_2 → neutral_3`。
- 实验 3 `neutral_info`：三条中性文本团队层面随机排列且不重复。
- 实验 3 `coop_narrative`：三条合作叙事文本团队层面随机排列且不重复，并与当段副线合作主题一致。
- `continuous / batch` 仍只控制提醒频率，不改变后台副线题目池、抽样组成与到达顺序。

团队级指导语计划保存到 `experimentSnapshot.instructionPlan`，随 session 创建时固定；admin 后续切换模式不影响旧 session。
