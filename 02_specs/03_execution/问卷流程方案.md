# 问卷流程方案

更新时间：2026-06-17

## 1. 当前口径

系统只承载正式实验内问卷，不承载招募问卷。师兄 HTML 中的计时、预览、论文来源、调试说明和页面样式都不进入平台；平台只提取正式题目文字、题型、选项、量表端点、是否必答、题目代码与构念字段。

被试前台只显示题目和作答控件，不显示题目代码、构念名、来源论文、链接或内部备注。

## 2. 问卷组成

正式问卷模板来自 `apps/server/src/questionnaire/three-chapter-v1-1.ts`，模板 ID 为 `three-chapter-questionnaire-v1-1`。

招募问卷：
- 来源 HTML 中存在 `recruitment` 页，但不接入系统。
- 后续如需招募筛选，由实验组另行发外部问卷。

工作段后问卷：
- 工作段 1 结束后呈现一次。
- 工作段 2 结束后呈现一次。
- 工作段 3 结束后，在最终结束页先补呈现一次。
- 三次使用同一套 `segmentSurvey` 题目，但保存到不同 `segmentIndex`。

最后问卷：
- 在第三次工作段后问卷提交后呈现。
- 组装方式为：共同模块 + 当前实验模式操纵检验 + 当前角色专属模块。
- `ai_upgrade` 使用实验 1 操纵检验。
- `side_reminder` 使用实验 2 操纵检验。
- `coop_narrative` 使用实验 3 操纵检验。
- `manual` 不额外插入实验 1/2/3 操纵检验模块。
- A 角色插入 A 专属题，B 角色插入 B 专属题。

## 3. 前端流程

`/break`：
- 在正式休息段读取 `runtime.questionnaireTemplate`。
- 呈现对应工作段后问卷。
- 提交后留在休息页等待倒计时结束。

`/workspace/end`：
- 如果还有未提交的第三次工作段后问卷，先呈现该问卷。
- 第三次工作段后问卷提交后，继续呈现最后问卷。
- 最后问卷提交后才显示实验完成页，并记录 `experiment_completed`。

## 4. 后端保存

所有正式问卷写入 `QuestionnaireResponse`：

| 问卷 | phase | segmentIndex | eventType |
|---|---:|---:|---|
| 工作段 1 后问卷 | `FORMAL` | 2 | `segment_survey_submitted` |
| 工作段 2 后问卷 | `FORMAL` | 4 | `segment_survey_submitted` |
| 工作段 3 后问卷 | `FORMAL` | 6 | `segment_survey_submitted` |
| 最后问卷 | `FORMAL` | 99 | `post_survey_submitted` |

`answers` JSON 包含：
- `kind`
- `templateVersion`
- `title`
- `experimentMode`
- `role`
- `segmentIndex`
- `workSegment`
- `submittedAt`
- `items`
- `answers`

其中 `items` 保存题目结构，便于后续核对当时题目版本；不包含论文来源、链接或师兄 HTML 里的说明性信息。

## 5. Admin 配置

admin 新增“问卷配置”tab：
- 展示工作段后问卷和最后问卷的组装说明。
- 支持编辑题干、选项、量表端点、开放题最大字数。
- 保存时写回 `QuestionnaireTemplate.items`。

旧“休息问卷模板”不再放在“实验配置”tab 中，避免与正式三章问卷并存造成误解。

## 6. 导出

导出包中问卷目录为：

```text
questionnaires/
  practice_quiz.json
  segment_1.json
  segment_2.json
  segment_3.json
  post_survey.json
```

`variables.json.questionnaire` 摘要字段：
- `practiceQuizPassed`
- `segmentSurveySubmittedCount`
- `segment1Submitted`
- `segment2Submitted`
- `segment3Submitted`
- `postSurveySubmitted`

不导出招募问卷，不导出论文来源或链接。
